## 1. 产品概述

### 1.1 产品愿景

学AI是一个面向中文用户的AI技术在线学习平台，致力于降低AI技术的学习门槛，为不同水平的学习者提供系统化、实战化的AI课程体系。平台覆盖LLM、RAG、Agent、Prompt Engineering、微调、计算机视觉、NLP、AI安全、模型部署、AI工具、多模态AI、AI基础等12大技术方向，帮助学习者从入门到精通，构建完整的AI技术能力。

### 1.2 目标用户

| 用户类型 | 描述 |
|----------|------|
| AI初学者 | 对AI感兴趣但缺乏系统学习路径的零基础用户 |
| AI从业者 | 希望深入学习特定AI技术方向的工程师和研究人员 |
| 转型开发者 | 从传统软件开发转向AI方向的技术人员 |
| 企业管理者 | 需要了解AI技术以做出技术决策的管理层 |
| 学生群体 | 计算机科学、人工智能相关专业的在校学生 |

### 1.3 核心价值主张

- **体系化课程**: 覆盖12大AI技术方向，从入门到高级，提供完整的学习路径
- **实战导向**: 课程包含大量动手实验、编码练习和项目实战
- **灵活付费**: 提供免费课程和付费课程，支持单课购买和VIP会员
- **学习激励**: 通过成就系统、连续打卡、经验值等机制激励持续学习
- **认证体系**: 提供课程考试和证书认证，验证学习成果
- **中文原生**: 全中文内容，贴合国内开发者的学习习惯和技术生态

### 1.4 产品架构

```
学AI平台
├── 主应用 (端口 3000) - 面向用户的学习平台
│   ├── 课程浏览与购买
│   ├── 在线学习与进度追踪
│   ├── 考试认证系统
│   ├── 个人中心与成就系统
│   └── 博客内容系统
│
└── 管理后台 (端口 3001) - 面向运营的管理平台
    ├── 数据仪表盘
    ├── 用户管理
    ├── 课程管理
    ├── 订单管理
    ├── 内容管理
    └── 系统设置
```

## 5. 安全需求

基于代码审计发现的安全问题，以下安全需求按优先级排列。

### 5.1 P0 - 必须立即修复

| 编号 | 问题 | 当前状态 | 修复方案 | 验收标准 |
|------|------|----------|----------|----------|
| S-001 | **硬编码密钥** | Yipay密钥、邮箱授权码硬编码在源码中作为fallback默认值 | 所有敏感配置必须从环境变量读取，缺失时拒绝启动 | 代码中无任何硬编码密钥 |
| S-002 | **课时内容API未鉴权** | `/api/lessons/[id]` 无认证检查，付费课程内容可被直接访问 | 付费课时需验证用户登录状态及购买记录 | 未购买用户无法获取付费课时内容 |
| S-003 | **邮件验证码存储在内存** | `Map`存储验证码，进程重启丢失，且无容量限制 | 迁移至Redis存储，设置TTL，限制存储容量 | 验证码持久化且有容量保护 |
| S-004 | **缺少API速率限制** | 注册、登录、发送验证码等接口无频率限制 | 实施基于IP的速率限制（如每分钟10次） | 防止暴力破解和资源滥用 |
| S-005 | **支付回调缺少幂等性保护** | 支付通知处理无并发控制 | 使用数据库事务+状态检查实现幂等 | 并发回调不会导致重复开通 |

### 5.2 P1 - 尽快修复

| 编号 | 问题 | 当前状态 | 修复方案 | 验收标准 |
|------|------|----------|----------|----------|
| S-006 | **JWT secret可能未配置** | `process.env.NEXTAUTH_SECRET` 可能为空 | 启动时校验环境变量，缺失则拒绝启动 | 所有必要环境变量启动时校验 |
| S-007 | **管理后台API鉴权不一致** | 部分管理API缺少权限检查 | 统一中间件强制校验admin角色 | 所有管理API均有权限校验 |
| S-008 | **输入校验不足** | API请求体缺少严格的schema校验 | 使用Zod对所有API入参进行校验 | 所有API入参均有schema校验 |
| S-009 | **密码强度要求低** | 仅要求6位字符 | 至少8位，包含大小写字母和数字 | 注册和修改密码时强制校验 |
| S-010 | **缺少CORS配置** | 未配置跨域资源共享策略 | 配置允许的域名白名单 | 仅允许指定域名跨域访问 |

### 5.3 P2 - 持续改进

| 编号 | 问题 | 当前状态 | 修复方案 | 验收标准 |
|------|------|----------|----------|----------|
| S-011 | **缺少请求日志** | 仅有console.error | 结构化日志记录所有API请求 | 关键操作可追溯 |
| S-012 | **缺少CSRF防护** | 依赖NextAuth默认配置 | 显式配置CSRF token | 关键操作有CSRF防护 |
| S-013 | **数据库连接无池化限制** | Prisma默认配置 | 配置连接池参数 | 高并发下数据库稳定 |
| S-014 | **错误信息泄露** | 部分错误返回堆栈信息 | 生产环境仅返回通用错误信息 | 用户看不到技术细节 |
| S-015 | **缺少安全响应头** | 未配置安全HTTP头 | 添加X-Frame-Options、CSP等 | 安全扫描通过 |

## 6. 数据模型概览

### 6.1 核心实体关系

```
用户系统
  User (用户)
    ├── 1:N → Enrollment (报名)
    ├── 1:N → Order (订单)
    ├── 1:N → UserProgress (学习进度)
    ├── 1:N → ExamAttempt (考试记录)
    ├── 1:N → Certificate (证书)
    ├── 1:N → UserAchievement (成就)
    ├── 1:N → UserStreak (打卡记录)
    ├── 1:N → Comment (评论)
    └── 1:N → Notification (通知)

课程系统
  Category (分类)
    └── 1:N → Course (课程)
  Course (课程)
    ├── 1:N → Module (模块)
    ├── 1:N → Enrollment (报名)
    ├── 1:N → Order (订单)
    ├── 1:N → Exam (考试)
    └── 1:N → Comment (评论)
  Module (模块)
    └── 1:N → Lesson (课时)
  Lesson (课时)
    ├── 1:N → UserProgress (进度)
    ├── 1:N → Quiz (测验题)
    └── 1:N → Comment (评论)

支付系统
  Order (订单)
    └── 1:1 → Payment (支付记录)

考试系统
  Exam (考试)
    ├── 1:N → ExamAttempt (考试记录)
    └── 1:N → Order (订单)

成就系统
  Achievement (成就定义)
    └── 1:N → UserAchievement (用户成就)
```

### 6.2 实体说明

| 实体 | 说明 | 关键字段 |
|------|------|----------|
| User | 平台用户 | email, password, role, vipLevel, exp, coins, streak |
| Category | 课程分类 | name, slug, icon |
| Course | 课程 | title, slug, price, isFree, level, status, tags |
| Module | 课程模块 | title, courseId, sortOrder |
| Lesson | 课时 | title, type(video/text/quiz/code), content, duration |
| UserProgress | 学习进度 | userId, lessonId, status, watchTime |
| Enrollment | 课程报名 | userId, courseId, progress |
| Order | 订单 | orderNo, userId, amount, status, paymentMethod |
| Payment | 支付记录 | orderId, tradeNo, type, status |
| Quiz | 测验题 | lessonId, question, type, options, answer |
| Exam | 考试 | title, courseId, passScore, timeLimit |
| ExamAttempt | 考试记录 | userId, examId, score, passed |
| Certificate | 证书 | userId, certificateNo, title |
| Achievement | 成就定义 | name, condition, expReward |
| UserAchievement | 用户成就 | userId, achievementId |
| UserStreak | 打卡记录 | userId, date, lessonsCompleted |
| Comment | 评论 | userId, lessonId/courseId, content |
| Notification | 通知 | userId, type, title, content, read |
| BlogPost | 博客文章 | title, slug, content, status |
| SystemSetting | 系统配置 | key, value |

## 8. 风险与对策

### 8.1 技术风险

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| SQLite性能瓶颈 | 高并发下数据库响应变慢 | 中 | 预留PostgreSQL迁移路径；优化查询；添加索引 |
| Next.js版本兼容性 | Next.js 16可能存在不稳定API | 中 | 锁定版本；关注官方更新；编写兼容层 |
| Yipay支付稳定性 | 第三方支付服务中断 | 低 | 备选支付渠道；支付状态轮询机制 |
| 验证码存储丢失 | 内存存储重启丢失 | 高 | 尽快迁移至Redis；增加持久化备选方案 |

### 8.2 安全风险

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| 密钥泄露 | 支付和邮件服务被滥用 | 高 | 立即移除硬编码；环境变量管理；密钥轮换 |
| API未授权访问 | 付费内容被免费获取 | 高 | 统一鉴权中间件；API权限矩阵 |
| 暴力破解 | 用户账号被破解 | 中 | 速率限制；账户锁定；验证码机制 |
| 支付漏洞 | 订单被篡改或重复处理 | 中 | 签名验证；幂等处理；金额校验 |

### 8.3 业务风险

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| 内容质量不达标 | 用户流失 | 中 | 建立内容审核机制；用户反馈闭环 |
| 用户增长缓慢 | 收入不达预期 | 中 | 免费课程引流；SEO优化；社群运营 |
| 竞品竞争 | 市场份额被抢占 | 中 | 差异化定位；持续更新内容；社区建设 |
| 内容版权纠纷 | 法律风险 | 低 | 原创内容为主；注明来源；版权声明 |

### 8.4 运维风险

| 风险 | 影响 | 概率 | 对策 |
|------|------|------|------|
| 服务器宕机 | 平台不可用 | 低 | 自动备份；监控告警；故障转移方案 |
| 数据丢失 | 用户数据和学习记录丢失 | 低 | 每日自动备份；异地备份；恢复演练 |
| 部署失败 | 新版本无法上线 | 中 | CI/CD流水线；回滚机制；灰度发布 |
