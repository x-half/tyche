# 🎯 日常使用指南

> 掌握这些技巧，让 OpenCode 成为你最高效的编程伙伴。

---

## 基础操作

### 启动
```bash
opencode
```

### 常用快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+C` | 中断当前操作 |
| `Ctrl+D` | 退出 |
| `↑` / `↓` | 查看历史命令 |
| `Tab` | 自动补全 |

## 怎么写好提示词

### 好的提示词 = 清晰 + 具体

❌ **不好的：**
```
帮我写个登录页面
```

✅ **好的：**
```
帮我写一个登录页面，React + TypeScript + Tailwind CSS。
要求：
1. 邮箱和密码输入框
2. 表单验证
3. 调用 /api/login 接口
4. 加载状态和错误提示
```

### 分步引导

复杂的任务让 AI 分步完成：

```
第一步：设计用户表结构
第二步：创建注册 API
第三步：写前端登录页面
第四步：编写测试
```

### 常见提示词

```
> 用 Python 写一个冒泡排序
> 解释这段代码 [粘贴代码]
> 优化这个函数 [粘贴代码]
> 为这个组件写测试 [粘贴代码]
> 帮我修复这个 bug [粘贴代码]
```

## 切换模型

在 opencode 中输入：

```
/models
```

选择你想要的免费模型。如果某个模型太慢，换一个试试。

---

## 实用命令

| 命令 | 功能 | 使用场景 |
|------|------|----------|
| `/help` | 查看帮助 | 记不住命令时 |
| `/models` | 切换模型 | 想换个 AI 模型 |
| `/info` | 查看会话信息 | 想知道当前状态 |
| `/reset` | 重置对话 | 想重新开始 |
| `/exit` | 退出 | 结束会话 |

## 工作流程

### 新项目

```bash
# 让 AI 帮你初始化项目
opencode "帮我创建一个 Next.js 项目"

# 添加功能
> 添加用户登录功能

# 写测试
> 为登录功能写测试
```

### 修改代码

```bash
# 打开项目
cd my-project
opencode

# 让 AI 理解现有代码
> 看看这个项目的代码结构

# 修改
> 把用户列表改为虚拟滚动
```

### 调试

```bash
> 页面报错 'Cannot read properties of undefined'，帮我看看
```

## 小技巧

| 技巧 | 说明 |
|------|------|
| **一次一件事** | 每次只说一个需求，别让 AI 混淆 |
| **说清楚上下文** | 告诉 AI 你用的是什么技术栈 |
| **逐步优化** | 先让 AI 写出基础版，再要求改进 |
| **及时反馈** | 对 AI 的输出说"好"或"修改这里" |
